META DATA USE STATEMENT FOR META oEMBED READ
============================================

Application: 3D Hiroba
App ID: 26224957440539490
Purpose: Display official Instagram embeds for curated community posts.

---

OVERVIEW
--------
3D Hiroba is a community gallery platform for sharing 3D pen creations.
Administrators curate publicly available Instagram posts as featured
items. Every featured post is rendered as the official Instagram embed —
both in the gallery grid card and in the detail view — exactly as
Instagram provides it. We do not cache thumbnails, do not store any
images from Instagram, and do not modify the embed's visual elements.

---

SUBMISSION FORM USE CASE DESCRIPTION (verbatim)
-----------------------------------------------

3D Hiroba is a community gallery for children sharing 3D pen creations.
Administrators curate publicly available Instagram posts as featured
items. For each featured post we call the Meta oEmbed Read endpoint
exactly once to obtain the official embed HTML (with hidecaption=true
and omitscript=true), and we render that HTML unchanged as the gallery
card itself — Instagram's own embed iframe loads in the visitor's
browser and the post image, profile, and "View on Instagram" link come
straight from www.instagram.com. We do not download, cache, or
otherwise persist any image, caption, author profile, or engagement
data from Instagram. The only Instagram-derived data we store is the
embed HTML string returned by Meta oEmbed Read, kept alongside the
canonical permalink so the embed can be re-rendered on subsequent page
loads; both are deleted immediately when the administrator removes the
pick. We do not combine Instagram data with other datasets, do not use
it for advertising, and do not share it with third parties.

---

DATA COLLECTED
--------------

From the Meta oEmbed Read endpoint
(graph.facebook.com/v19.0/instagram_oembed):
- html  - the official Instagram embed HTML used as the gallery card.

Parameters sent on every request:
- url=<canonical Instagram permalink>
- access_token=<APP_ID>|<APP_SECRET>
- omitscript=true     (embed.js loaded once from www.instagram.com)
- hidecaption=true    (caption hidden per administrator preference)
- fields=html

We do NOT request, parse, or persist any of:
- author_name / author_url
- thumbnail_url / thumbnail_width / thumbnail_height
- title
- engagement counts (likes, comments)
- profile data
- post image binaries

The post image, author profile, and "View on Instagram" link are loaded
by Instagram's own embed.js directly from www.instagram.com in the
visitor's browser. None of this traffic passes through our servers.

---

DATA USE & PURPOSE
------------------

PRIMARY USE: Render the official Instagram embed for each featured post.
- The gallery grid card IS the official Instagram embed iframe.
- The detail modal also renders the same official embed (slightly
  larger viewport).
- We never reproduce Instagram content in a custom layout.

NOT USED FOR:
- Targeted advertising.
- User profiling or tracking.
- Selling or sharing with third parties.
- Training machine learning models.
- Re-identification of users.
- Combining Instagram data with other datasets.

---

DATA STORAGE & SECURITY
-----------------------

curation.json on Vercel Blob (admin curation metadata)
- pick id, administrator-entered title and pentaComment, the canonical
  permalink, the embed HTML returned by Meta oEmbed Read, addedAt
  timestamp.
- Written only by the admin API; read by the public site.

No other Instagram-derived data is stored. There is no thumbnail cache,
no image binary stored on our side, no scheduled refresh job.

Security
- HTTPS only.
- Admin actions gated by httpOnly + Secure auth cookie.
- Server-side validation: shortcode regex extraction, canonical
  permalink normalization (https://www.instagram.com/p/<shortcode>/).

---

DATA RETENTION & DELETION
--------------------------

Retention
- The embed HTML and permalink are kept while the post is featured.
- Removing a pick from the admin dashboard immediately deletes the
  curation entry.

User Rights
- Anyone can request deletion via help@scrib3dpen.jp or /contact.
- We honor takedown requests within 24 hours of confirmation.

---

THIRD-PARTY SHARING
--------------------

No third party receives Instagram-derived data from us.
- Vercel hosts the application (processor role).
- Meta receives the post URL and our app token when we call
  oEmbed Read.

API USAGE PATTERN
-----------------

1. Admin pastes an Instagram URL into the management interface.
2. Server normalizes it to https://www.instagram.com/p/<shortcode>/.
3. Server calls Meta oEmbed Read once with hidecaption=true and stores
   the returned html string.
4. The pick is shown in the public gallery as the official Instagram
   embed iframe (rendered via embed.js loaded from www.instagram.com).
5. Pick deletion removes the curation entry.

Administrators may also paste an Instagram embed code obtained from
Instagram's own share dialog; in that case we store the pasted HTML
without calling Meta oEmbed Read for that pick.

---

CHILDREN & MINORS
-----------------

3D Hiroba targets families with children aged 5-14. The gallery shows
only public Instagram content curated by adult administrators, rendered
as Instagram's own official embed. No accounts, no sign-in for
children, no profiling of minors.
